

## Recent Understandings - 2026-03-09

### Agent Delegation Limitations (Critical)

- **Subagents cannot use `sessions_spawn`**: Despite configuration (`tools.agentToAgent.enabled: true`, `tools.agentToAgent.allow`, agent-level `tools.allow`), the OpenClaw runtime explicitly prevents subagents from spawning other agents. This is a fundamental runtime restriction, not a misconfiguration.
- **`subagents(action='steer')` behavior**:
    - Steer sends a message to a subagent's session.
    - The target subagent receives it as a new message and processes it.
    - The steering agent (e.g., Bernard) waits for a response.
    - **Crucially**: `subagents(action='list')` only shows agents that are *children* of the current agent.
- **No Peer-to-Peer Communication**: Because subagents cannot spawn other agents, they cannot create the parent-child relationships necessary for `subagents(action='steer')` to enable true peer-to-peer communication between agent *siblings*. All inter-agent communication must flow through the main agent (Claw).

### Revised Agent Architecture
- **Claw is the Sole Spawner**: Claw is the only agent with `sessions_spawn` capability.
- **Flat Orchestration**: The effective architecture is: `Brett → Claw (spawns all agents) → All agents (siblings, cannot talk directly to each other)`.
- **Bernard's Role Re-evaluated**: With these limitations, Bernard's role as a coordinator who delegates by spawning or directly interacting with other agents is significantly reduced. His value is primarily in complex strategic planning leveraging his model, or synthesizing results if Claw explicitly forwards them to him.

### Proposed Next Steps:
1.  **Simplify Workflow**: For most operational tasks, Claw should directly spawn specialist agents (Devan, Vale, etc.) and coordinate their work.
2.  **Bernard's Niche**: Bernard should be reserved for high-level strategic planning or deep synthesis tasks, where his model offers unique value.
3.  **Investigate Nested Spawning**: Explore if there's any undocumented way or future feature to allow nested subagent spawning (agents as children of other agents), which would enable Bernard to use `subagents(action='steer')` more effectively with his team. (Currently, no obvious `sessions_spawn` parameters exist for this.)
