### Brett's Request

**"Just make sure I get notified when phases get finished."**

Simple ask. Critical need. Previously had no systematic approach.

### The Problem

**Current state:**
- Agents finish work silently
- Bernard sees completions on next heartbeat (up to 20 min delay)
- Brett doesn't get notified unless he asks
- Completions could sit for hours without visibility

### Solution Implemented

**Created `COMPLETION_NOTIFICATION_PROTOCOL.md`** with three-layer safety net:

**Layer 1 (Primary):** Agent announces directly
- When task completes, agent includes notification in final response
- "@Brett - Phase X complete and ready for review"
- Delivers immediately via Telegram

**Layer 2 (Backup):** Bernard's heartbeat check
- Every 20 minutes, scans for COMPLETE tasks
- Checks if "Notified: [timestamp]" exists
- If missing >5 min, alerts Claw to notify Brett

**Layer 3 (Fallback):** BULLETIN_BOARD.md visibility
- Brett can check anytime
- Status dashboard shows completions
- Timestamp tracking

### Agent Completion Template

**When marking task COMPLETE, agents must end with:**

```
---

✅ Task #XXX Complete!

**Deliverables:**
- [List what was created/done]
- [Links to files/reports]

**Status:** Ready for review
**Next:** [What happens next]

@Brett - Phase X is complete and ready for your review.
```

### Files Updated

1. **`COMPLETION_NOTIFICATION_PROTOCOL.md`** (3KB) — Complete protocol
2. **Bernard's `HEARTBEAT.md`** — Added completion checking (step 2)
3. **System event sent** — Bernard notified of new protocol

### Expected Behavior

**When Vale finishes Phase 6:**
1. Vale updates BULLETIN_BOARD.md: "✅ COMPLETE"
2. Vale's response: "@Brett - Phase 6 complete! Ready for review."
3. **Brett gets Telegram notification immediately**
4. Bernard confirms on next heartbeat (backup)

**If somehow missed:**
- Bernard spots COMPLETE without "Notified: [timestamp]"
- Alerts Claw with summary
- Claw delivers notification to Brett
- Timestamp added to prevent duplicate alerts

### Why Multiple Layers

**Redundancy prevents information loss:**
- Primary fails → Backup catches it
