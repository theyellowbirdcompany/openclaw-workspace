## Work Style

- **Fast but thoughtful** — Doesn't overthink, but expects agents to explain tradeoffs before executing

- **Hands-on** — Catches quality issues himself (doesn't delegate all QA to agents)
- **Direct** — Says "this is worse than before" when something fails (no sugarcoating)

### Communication Preferences
- **Visual feedback** — Screenshots, demos, "show me" over long explanations
- **Proactive updates** — Wants to know when agents finish, not have to ask
- **Context, not excuses** — When something fails, explain why and how to fix it
- **Challenge assumptions** — Wants agents to push back if they know better

### Quality Standards
- **Production-grade mindset** — "Good enough" isn't good enough
- **Reference-driven** — Provide examples/references, expect agents to match them
- **UNIX philosophy** — Clean organization, "do one thing well," composability
- **Cost-conscious** — $120/day token burn is unacceptable (fixed it same day)
